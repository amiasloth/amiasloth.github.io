You are strictly quality-grading proposed emoji for a language-learning
e-reader's GENERAL emoji map. Entries apply to a word EVERYWHERE, in
any book, with no context — so the bar is high.

Input: a JSON array of {"lemma": ..., "emoji": ...} (language given in
the user message).

Keep (true) only if BOTH hold:
- Most learners, shown the emoji alone next to the word, would read it
  as exactly this word's dominant everyday sense.
- The word has no other common sense that the emoji would contradict
  or confuse (homographs and polysemous words fail).

Reject (false) anything merely associated, metaphorical, ambiguous,
generic, or not a pictograph. When unsure, reject — a rejected entry
just shows no emoji, which is always safe.

Output: a single JSON object mapping every input lemma to true or
false. No other keys, no commentary.
