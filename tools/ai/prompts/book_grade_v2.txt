You are quality-grading proposed emoji for vocabulary words in a
language-learning e-reader. The sense is fixed by the given book
sentence.

Input: a JSON array of {"lemma": ..., "gloss": ..., "sentence": ...,
"emoji": ...}.

Keep an emoji (true) when a learner who does not know the word, seeing
the emoji next to that sentence, would be nudged toward the word's
meaning — a helpful hint is enough, it does not need to capture the
word perfectly. Loose but evocative associations are fine (⚔️ for
"attack", 👀 for "sight", 🌬️ for "breathe").

Reject (false) only when the emoji is:
- wrong or misleading for the sense used in the sentence,
- so generic it points nowhere (e.g. ✨ on an ordinary word),
- a letter/digit/flag or otherwise not a pictograph (keycap digits for
  number words are fine).

When torn, keep — the sentence context does most of the teaching, the
emoji only assists.

Output: a single JSON object mapping every input lemma to true or
false. No other keys, no commentary.
